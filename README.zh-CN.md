# Mini Action Task（中文说明）

[English](./README.md)

Mini Action Task 是一个离线优先的 Flutter 任务管理应用，强调“先行动，再优化”。  
它通过“下一步动作”拆解、能量匹配与数据复盘，帮助你持续推进任务而不是反复拖延。

## 这个项目解决什么问题

- 降低启动成本，让任务更容易开始。
- 在低状态时也能保持最小连续推进。
- 兼顾长期目标与日常执行节奏。
- 用可视化复盘持续优化行动习惯。

## 核心功能

- **任务生命周期管理**：支持待选、进行中、已完成、冻结、删除等状态管理。
- **下一步动作拆解**：可批量输入动作（每行一个），把复杂任务拆成可执行步骤。
- **能量感知规划**：为任务设置 0.0-5.0 能量值，并标记“低能量可执行”。
- **智能推荐任务**：基于当前上下文与近期能量状态推荐更适合的任务。
- **自动冻结保护**：长时间未完成的任务可自动冻结，缓解任务堆积压力。
- **行为日志记录**：自动记录任务相关动作，便于回顾和衔接上下文。
- **统计与趋势视图**：查看完成趋势与动作热力分布，识别执行节奏。
- **个人辅助能力**：支持个人信息配置、提醒语录、数据备份与导入导出。
- **离线优先**：所有数据本地存储，无网络也能稳定使用。

## 任务字段模型

每个任务可包含以下关键属性：

- **重要性**：主线 / 日常 / 习惯
- **优先级**：用于安排执行顺序
- **能量预估**：完成该任务所需精力
- **低能量可执行**：是否适合状态较低时处理
- **截止窗口**：距离截止日期剩余天数
- **下一步动作**：推动任务前进的具体可执行步骤

## 技术栈

- **框架**：[Flutter](https://flutter.dev/)
- **数据持久化**：本地 SQLite
- **标识策略**：UUID
- **代码结构**：models / services / screens / widgets 分层

## 快速开始

### 环境要求

- Flutter SDK（稳定版）
- Android Studio 或 VS Code（含 Flutter/Dart 插件）
- Android/iOS 模拟器或真机

### 本地运行

1. 克隆仓库。
2. 安装依赖：
   ```bash
   flutter pub get
   ```
3. 启动应用：
   ```bash
   flutter run
   ```

## 适合谁使用

- 想快速启动任务、减少拖延的人。
- 习惯把目标拆解为行动清单的人。
- 希望通过轻量化离线复盘持续优化执行力的人。
